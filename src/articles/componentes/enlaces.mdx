## Tipos de enlace

Los enlaces facilitan la accesibilidad a contenido adicional y aseguran que las personas usuarias puedan comprender y utilizar efectivamente las rutas de navegación propuestas. Los enlaces pueden estar dentro de un párrafo o separados del mismo en una estructura de lista.
<br />

![completar](/componentes/enlace/tiposDeEnlaces.svg)

### Predeterminado

Se utiliza para enlazar contenido que se encuentra dentro del mismo sitio. Se puede utilizar para navegar dentro de una misma página como ancla, o para correos electrónicos y direcciones. 

<div class="text-center pt-5 pb-3">

  ![completar](/componentes/enlace/Enlace-predeterminado.svg)

</div>

### Externo

Se utiliza para enlazar contenido a un sitio web externo por fuera del dominio actual. 

<div class="text-center pt-5 pb-3">

  ![Enlace externo](/componentes/enlace/enlace-externo.svg)

</div>

### De acceso

Se utiliza para ver más información dentro de una sección de un mismo sitio. Por ejemplo, "Ver más noticias".

<div class="text-center pt-5 pb-3">

  ![completar](/componentes/enlace/Enlace-de-acceso.svg)

</div>

### De descarga

Se utiliza para indicar la descarga de un recurso.

<div class="text-center pt-5 pb-3">

  ![completar](/componentes/enlace/Enlace-de-descarga.svg)

</div>

## Diseño y estructura (contexto de uso)

Los enlaces cuentan con dos tamaños (grande y chico) para desktop y mobile, y con cuatro estados: predeterminado, sobre, en foco y visitado. Se pueden colocar dentro y fuera de un párrafo de texto, y agruparse dentro de una <NoRedirectLink textLink="Lista de enlaces" />.

## Usabilidad

### Buenas prácticas

[[si | El texto de un enlace tiene que ser conciso y explicativo. Debe representar su propósito con claridad de modo que se entienda el destino del vínculo.]]

[[si | Incluir los enlaces al comienzo o final de las oraciones para facilitar la lectura.]]

[[si | Elegir el tipo de enlace adecuado según la navegación propuesta. ]]

### Malas prácticas

[[no | No utilizar demasiados enlaces dentro de un párrafo. Los enlaces agregan carga cognitiva y afectan a la lectura. Se sugiere limitarse a un enlace por oración para simplificar el mensaje.]]

[[no | No utilizar los enlaces como botones porque cumplen funciones diferentes. Los botones son llamados a la acción y los enlaces son hipervínculos.]]

[[no | No usar enlaces con textos ambiguos «click aquí » o «más información»; la persona usuaria debe saber exactamente adónde se dirigirá al hacer click el enlace.]]

[[no | No utilizar textos de enlace en negritas. Tampoco utilizar signos de puntuación.]]

[[no | No escribir TODO EN MAYÚSCULAS; dificulta la lectura y por convención web, sería como gritar o levantar la voz. Para más información ver la sección de <a href="https://gcba.github.io/estandares/contenido/comunicar-con-claridad/" target="_blank">Contenidos</a>.]]

[[no | No quitar el ícono que acompaña a los enlaces externos, de acceso y de descarga. Se utilizan como refuerzo semántico del tipo de enlace.]]
