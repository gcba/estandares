## Tipos de campos de un formulario

### Campo de texto simple

Los campos de texto simples pueden llevar un placeholder para mostrar un ejemplo de los datos que se están solicitando. El área de ayuda proporciona información de asistencia extra.

<div class="content-2">
  <div class="form-wrapper">
    <div class="form-group">
      <label for="name-input">Nombre</label>
      <input type="text" class="form-control" id="name-input" name="name" />
    </div>
  </div>
  <div class="form-wrapper bg-light p-4">
    <div class="form-group">
      <label for="name-input">Nombre</label>
      <input type="text" class="form-control" id="name-input" name="name" />
    </div>
  </div>
</div>

### Selectores

Los selectores se utilizan cuando el sistema ya posee los datos que se precisan y el usuario solo debe seleccionar uno de ellos. Debe haber al menos cinco opciones de selección.

<div class="content-2">
  <div class="form-wrapper">
    <div class="form-group">
      <label for="lista-de-seleccion">Nombre del selector *</label>
      <select class="custom-select" id="lista-de-seleccion">
        <option>1</option>
        <option>2</option>
        <option>3</option>
        <option>4</option>
        <option>5</option>
      </select>
    </div>
  </div>
  <div class="form-wrapper bg-light p-4">
    <div class="form-group">
      <label for="lista-de-seleccion">Nombre del selector</label>
      <select class="custom-select" id="lista-de-seleccion">
        <option>1</option>
        <option>2</option>
      </select>
    </div>
  </div>
</div>

### Campo de búsqueda

Los campos de búsqueda llevan siempre un placeholder que anticipe y ayude al usuario a saber qué información podrá encontrar en el buscador.

  </div>
  <div class="form-wrapper bg-light p-4">

### Campo de fecha

Los campos de fecha siempre llevan un placeholder que indique al usuario cómo debe completar el campo, ejemplo: dd/mm/aaaa. Cuando se acciona el campo la fecha se completa mediante el teclado.

<div class="content-2">
  <div class="form-wrapper">
    <div class="form-group">
      <label for="date-input">Fecha</label>
      <span id="date-help" class="form-text">
        Nombre del campo *
      </span>
      <input type="date" class="form-control" id="date-input" name="date" aria-describedby="date-help" />
    </div>
  </div>
  <div class="form-wrapper bg-light p-4">
    <div class="form-group">
      <label for="date-input">Fecha</label>
      <span id="date-help" class="form-text">
        Nombre del campo *
      </span>
      <input type="date" class="form-control" id="date-input" name="date" aria-describedby="date-help" />
    </div>
  </div>
</div>

### Radio button

Permite seleccionar una sola opción de una lista de opciones. Lo recomendable es que el usuario pueda ver todas las opciones al mismo tiempo y compararlas.

import { RadioInput } from "../../../components/docs/forms/RadioInput"

<div class="content-2">
  <div class="form-wrapper d-grid text-center">
    <RadioInput
      name="skills"
      options={[
        { value: "uno", label: "Opción 1" },
        { value: "dos", label: "Opción 2" },
        { value: "tres", label: "Opción 3" },
      ]}
    />
  </div>
</div>

## Diseño y estructura (contexto de uso)

Un formulario permite a los usuarios ingresar información en una página.

Los botones deben estar ordenados por importancia de derecha a izquierda, con el botón más importante (principal) a la derecha.

Agrupar las secciones relacionadas en conjuntos lógicos para que los usuarios puedan comprender la información que necesitan ingresar.

## Usabilidad

### Buenas prácticas

[[si | **Reducir la cantidad de campos del formulario** para evitar una mayor carga cognitiva en los usuarios.]]

[[si | **Evitar los campos opcionales** solicitando a los usuarios solo información relevante.]]

[[si | **Colocar etiquetas en los campos de texto.** Un campo sin etiqueta es ambiguo y poco accesible.]]

[[si | **Utilizar campos adecuados a la longitud del texto requerido.** La longitud del campo de texto proporciona a los usuarios una pista sobre la cantidad de texto que deben escribir.]]

[[si | **Colocar etiquetas seleccionables en los checkbox y radio buttons.** Los usuarios deben poder tocar o hacer clic en la etiqueta de texto o en el ícono para realizar una selección.]]

### Malas prácticas

[[no | **Evitar deshabilitar el botón de envío,** incluso si no se completaron todos los campos obligatorios. En su lugar, describir lo que debe hacerse usando mensajes de error y validación.]]

[[no | **Evitar posicionar los checkbox y radio buttons de forma horizontal.** Los listados con esa direccionalidad pueden dificultar saber qué etiqueta corresponde a cada opción.]]
