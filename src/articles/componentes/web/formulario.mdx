## Anatomía

<!-- ![completar](/componentes/alerta/anatomía-1.svg) -->

1. El label es una introducción al campo que hay que completar. 
2. El campo es el lugar donde el usuario tiene que ingresar los datos correspondiente que se piden en cada control. 
3. El ícono indica y acompaña la acción que se puede hacer en el tipo de campo. 
4. El área de ayuda es opcional. En caso de ser necesario se puede incluir un mensaje que ayude al usuario para completar el campo. En caso de ser un campo obligatorio y no estar completo, ese espacio se utiliza para un mensaje de error indicando que falta completar el campo. Funciona igual para todos los tipos de campo.

## Opciones de controles de formulario

### Campo de texto simple

Los campos de texto simples pueden llevar un placeholder para mostrar un ejemplo de los datos que se están solicitando. El área de ayuda proporciona información extra de asistencia.

<div className="form-wrapper">
  <div className="form-group">
    <label htmlFor="name-input">Nombre</label>
    <input type="text" className="form-control" id="name-input" name="name" placeholder="Ejemplo: Camila" />
  </div>
</div>

### Selector

Los selectores se utilizan cuando el sistema ya posee los datos que se precisan y el usuario solo debe seleccionar uno de ellos. Debe haber al menos cinco opciones de selección.

<div className="form-group">
  <label htmlFor="lista-de-seleccion">Example select</label>
  <select className="custom-select" id="lista-de-seleccion">
    <option>1</option>
    <option>2</option>
    <option>3</option>
    <option>4</option>
    <option>5</option>
  </select>
</div>

### Campo de búsqueda

Los campos de búsqueda llevan siempre un placeholder que anticipe y ayude al usuario a saber que información podrá encontrar en el buscador.

### Campo de fecha

Los campos de fecha siempre llevan un placeholder que indiquen al usuario como debe completar el campo. Cuando se acciona el campo, la fecha se completa mediante el teclado.

<div className="form-wrapper">
  <div className="form-group">
    <label htmlFor="date-input">Fecha</label>
    <input type="date" className="form-control" id="date-input" name="date" aria-describedby="date-help" />
    <small id="date-help" className="form-text text-muted">
      Ingrese una fecha posterior a hoy.
    </small>
  </div>
</div>

### Área de texto

El área de texto puede tener o no una limitación de caracteres. Cuando hay una limitación se visualiza con la cantidad de caracteres que hay actualmente y la cantidad máxima permitida.

<div className="form-wrapper">
  <div className="form-group">
    <label htmlFor="text-area">Bloque de texto</label>
    <textarea className="form-control" id="text-area" rows={3}></textarea>
  </div>
</div>

### Campo numérico

Los campos numéricos pueden cargarse tanto por teclado como por las flechas indicadoras. Este tipo de campos se utilizan cuando necesitamos un valor de cantidad.

<div className="form-wrapper">
  <div className="form-group">
    <label htmlFor="number-input">Edad</label>
    <input type="number" className="form-control" id="number-input" name="number" value="30" />
  </div>
</div>

### Radio button

Permite seleccionar una sola opción de una lista de opciones. Lo recomendable es que el usuario pueda ver todas las opciones al mismo tiempo y compararlas.

<div className="custom-control custom-radio">
  <input className="custom-control-input" type="radio" name="profession" id="profession-designer" value="designer" checked  />
  <label className="custom-control-label" htmlFor="profession-designer">
    Opción 1
  </label>
</div>
<div className="custom-control custom-radio">
  <input className="custom-control-input" type="radio" name="profession" id="profession-developer" value="developer" />
  <label className="custom-control-label" htmlFor="profession-developer">
    Opción 2
  </label>
</div>

### Checkbox

Permite seleccionar varios elementos de una lista de elementos individuales o marcar un elemento individual como seleccionado.

<div className="custom-control custom-checkbox">
  <input className="custom-control-input" type="checkbox" id="skills-copywrite-input" name="skills" value="copywrite" />
  <label className="custom-control-label" htmlFor="skills-copywrite-input">
    Opción 1
  </label>
</div>
<div className="custom-control custom-checkbox">
  <input className="custom-control-input" type="checkbox" id="skills-design-input" name="skills" value="design" />
  <label className="custom-control-label" htmlFor="skills-design-input">
    Opción 2
  </label>
</div>
<div className="custom-control custom-checkbox">
  <input className="custom-control-input" type="checkbox" id="skills-development-input" name="skills" value="development" />
  <label className="custom-control-label" htmlFor="skills-development-input">
    Opción 3
  </label>
</div>

## Usabilidad

**Campos adecuados a la longitud del texto requerido**. La longitud del campo de texto proporciona a los usuarios una pista sobre la cantidad de texto que deben escribir. 

**Usar un área de texto para que los usuarios escriban párrafos de textos extensos**. 

**Esperar para validar un campo**. Mostrar mensajes de validación de errores o estilos después de que un usuario haya interactuado con un campo en particular. 

**Evitar los menús desplegables en el campo de fecha**. Puede ser tentador cambiar todos o algunos de estos campos de texto a menús desplegables, pero estos tienden a ser más difíciles de usar que los cuadros de texto. 

**En el campo de fecha describir el formato**. Esto proporciona una ayuda a los usuarios al ingresar el formato de fecha adecuado. Ejemplo: dd/mm/aaaa 

**En los checkbox y radio buttons las etiqueta deben ser seleccionables**. Los usuarios deben poder tocar o hacer clic en la etiqueta de texto o en el ícono para realizar una selección. 

**Las opciones de los checkbox y radio buttons deben estar posicionadas verticalmente**. Los listados horizontales pueden dificultar saber qué etiqueta corresponde a cada opción.
