## Tipos de acceso

### Acceso sin borde

Se utilizan para componer organismos donde se establezcan múltiples accesos dentro de un mismo contenedor.

<div className="content-2">
  <div className="list-group">
    <a href="#" className="list-group-item item-sm no-shadow">
      <span className="access-title">Acceso</span>
    </a>
  </div>
</div>

### Acceso con borde

Se utilizan para acceder a secciones de mayor relevancia. Principalmente se usan en compañía de otros accesos generando agrupaciones de una misma temática.

<div className="content-2">
  <div className="list-group">
    <a href="#" className="list-group-item item-sm">
      <span className="access-title">Acceso</span>
    </a>
  </div>
</div>

### Acceso con descripción

La descripción es una variable del acceso para agregar información adicional que puede servirle a la persona usuaria a comprender mejor la acción a realizar.

<div className="content-2">
  <div className="list-group">
    <a href="#" className="list-group-item item-sm">
      <div className="access-content">
        <span className="access-title">Acceso</span>
        <p className="access-text">Descripción (Opcional)</p>
      </div>
    </a>
  </div>
</div>

### Acceso con ícono

El ícono sirve como referencia visual para complementar al título del acceso y así ayudar a la persona usuaria a tomar una decisión.

<div className="content-2">
  <div className="list-group">
    <a href="#" className="list-group-item item-sm">
      <i className="bx bxs-info-circle"></i>
      <div className="access-content">
        <span className="access-title">Acceso</span>
        <p className="access-text">Descripción (Opcional)</p>
      </div>
    </a>
  </div>
</div>

### Lista de accesos

El componente de acceso puede visualizarse tanto en posición horizontal como vertical. Para lograrlo, se ha implementado una combinación de clases llamadas "list-group" y "list-group-row", las cuales permiten organizar y adaptar el componente de forma adecuada en la página o aplicación. Además, se ha asegurado que el componente respete los tamaños de ancho y espaciado correspondientes para garantizar su apariencia y usabilidad.

## Usabilidad

### Buenas prácticas

[[si | Intentá incluir hasta 7 accesos en el organismo de “Acceso completo con lista de accesos” para que le sea más fácil a las personas usuarias de escanear con la vista todos los elementos.]]

[[si | Recomendamos limitarse a una línea de texto para el título y la descripción de los accesos.]]

### Malas prácticas

[[no | No utilizar puntos suspensivos para truncar el texto de los accesos]]
[[no | No darle funcionalidad a los accesos. Únicamente sirven para redirigir al usuario a otra sección o pantalla del sistema.]]
